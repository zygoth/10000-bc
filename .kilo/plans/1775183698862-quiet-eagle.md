## Fix Strategy

1. **Narrow display-complete semantics in tech forest helper**
   - Update `isTechResearchDisplayComplete` in `src/game/techForestGen.mjs` so it only marks a node complete when one of these is true:
     - strict ancestor chain is fully researched (normal researched path), or
     - the node itself has direct vision grant (`visionGranted[unlockKey] === true`), or
     - the node itself has partner research completion (`partnerResearched[unlockKey] === true`).
   - Do **not** allow descendant nodes to be implicitly complete due to ancestor vision state.

2. **Add/adjust unit tests for display logic**
   - Extend `src/game/techForestGen.test.js` with an explicit case:

     - Expect `isTechResearchDisplayComplete(..., child, sources) === false` until strict chain is satisfied or child is directly completed.
   - Keep existing positive tests for direct vision/partner-researched nodes intact.
